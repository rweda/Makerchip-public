Class: IDE_Plugin

Defined in: ../doc/cleaned_coffee_plugin/IDE_Plugin.coffee

Overview

Base class for third-party platform integration with the Makerchip IDE.

Extend this class to customize IDE behavior, manage code loading, and handle compilation events. The IDE is embedded in an iframe and communicates with your platform code via this API.

Internal use: See full class documentation at: https://makerchip.com/docs/#https://makerchip.com/docs/class/IDE_Plugin.html

Examples:

Single instance setup 

window.defineIDE = function(IDE_Plugin) {
  return class MyPlatform extends IDE_Plugin {
    onReady() {
      console.log("IDE ready!");
      // Enable UI, set up event handlers, etc.
    }
    onCodeChange() {
      // Save to localStorage, enable save button, etc.
    }
  };
};

Multiple instances on one page 

window.defineIDE = {};
window.defineIDE.circuit1 = function(IDE_Plugin) {
  return class Circuit1 extends IDE_Plugin {
    getDefaultCodeURL() { return 'examples/adder.tlv'; }
  };
};
window.defineIDE.circuit2 = function(IDE_Plugin) {
  return class Circuit2 extends IDE_Plugin {
    getDefaultCodeURL() { return 'examples/counter.tlv'; }
    initReadOnly() { return true; }
  };
};
// Then access via, e.g.: window.IDE_instances.circuit1.compile()

Instance Method Summary

  • # (String) getDefaultCodeURL() Get URL for default code to load on startup
  • # (Promise<String>) getDefaultCode() Get default code to load on startup or override getDefaultCodeURL() for simpler URL-based customization
  • # (Boolean) initReadOnly() Whether the initial code should be read-only
  • # (void) compile() Trigger compilation of current code in editor
  • # (void) setCode(code, readOnly = false) Set code in IDE editor
  • # (Promise) getCode() Get code from IDE editor
  • # (void) setCodeFromURL(url) Load code from URL
  • # (void|Promise) onReady() Third party entry point, called when IDE is fully loaded and ready.
  • # (void) onCodeChange() Called when user edits code in the editor (via typing, etc.) This is a notification only - no code is passed to avoid communication overhead.
  • # (void) onCodeReplaced() Called when code is programmatically replaced (via setCode, setCodeFromURL, loading examples, etc.) This is a notification only - no code is passed to avoid communication overhead.

# (void|Promise) onReady()

Note: Override to initialize and enable UI after IDE is ready

Third party entry point, called when IDE is fully loaded and ready. Can be async/return a Promise for async initialization - splash screen will not be removed until the Promise resolves, preventing user interactions during initialization.

Returns:

  • ( void|Promise ) — Optional Promise for async initialization

# (void) onCodeChange()

Note: Programmatic code replacements trigger onCodeReplaced, not this.
Note: Override to handle user edit notifications (e.g., save to localStorage)

Called when user edits code in the editor (via typing, etc.) This is a notification only - no code is passed to avoid communication overhead. Use getCode() if you need to retrieve the actual code.

See also:

  • onCodeReplaced

# (void) onCodeReplaced()

Note: This is mutually exclusive with onCodeChange - you get one or the other, never both
Note: Override to handle programmatic code replacement notifications

Called when code is programmatically replaced (via setCode, setCodeFromURL, loading examples, etc.) This is a notification only - no code is passed to avoid communication overhead. Use getCode() if you need to retrieve the actual code.
